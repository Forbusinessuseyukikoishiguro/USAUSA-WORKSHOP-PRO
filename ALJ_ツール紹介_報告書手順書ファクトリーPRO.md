# 報告書・手順書づくりを支える社内ツール「報告書＆手順書ファクトリー PRO」のご紹介

> IT教育の現場から。新人・受講生が「何を・どの順で書くか」で迷わないための、オフラインで動く単一HTMLツールをご紹介します。

## はじめに

研修や実務の現場で、新人や受講生がよくつまずくのが「報告書・手順書を、何を・どの順で書けばよいか分からない」という点です。書き方そのものが分からないために、内容以前のところで時間がかかってしまう——こうした声は、新人研修の現場で繰り返し聞かれます。

本記事でご紹介する「報告書＆手順書ファクトリー PRO」は、**項目を埋めるだけで、整った日本語のビジネス文書（報告書・手順書）が完成する**ツールです。各項目には「なぜそれを書くのか」というヒントが添えられており、文章の型を知らない方でも、型どおりの文書を作成できます。

結論から言えば、本ツールは次の3点を満たします。

- **インストール不要**：HTMLファイル1つをブラウザで開くだけで動作します。
- **オフライン対応**：作成・保存はインターネット接続なしで完結します。
- **AIは任意**：AIによる文章整形は補助機能であり、使わなくても本体は完全に動作します。

## このツールでできること

| 機能 | 内容 |
| --- | --- |
| 文書テンプレート | 報告書11種・手順書3種の計14種から選べます |
| 動的フォーム | 選んだタイプに応じて、入力欄とヒントが切り替わります |
| リアルタイム整形 | 入力すると同時に、右側に完成文がプレビュー表示されます |
| エクスポート | Markdown／Word／PDF／CSV／Excel の5形式で書き出せます |
| サンプル投入 | タイプ別の下書きを一括で入力し、書き出しの参考にできます |
| AI整形（任意） | 敬語の調整・短く／長く・字数調整などを補助します |

## 使い方の流れ

操作はとてもシンプルです。画面は大きく3つのエリアに分かれています。

1. **左：記入フォーム** … 件名・結論・実施内容などの項目を入力します。
2. **右：完成プレビュー** … 入力に合わせて、整った文書がその場で組み上がります。
3. **下：保存メニュー** … 完成した文書を、必要な形式で書き出します。

入力していない項目は出力に表示されません。そのため「書ける所だけ書く」という使い方ができ、空欄の見出しだけが残ってしまうこともありません。手順書では、手順が自動で番号付けされるなど、提出物として整いやすい工夫が施されています。

## 研修・実務での活かし方

本ツールは、単に文書を作るだけでなく、**「良い文書の型」を体験しながら身につける**教材としても活用できます。

- **新人研修での導入**：報告書の構成（結論→事実→所感→依頼など）を、穴埋め形式で体験できます。ヒントが常に表示されるため、「なぜこの順で書くのか」を理解しながら作成できます。
- **日報・週報の定着**：毎回ゼロから書く負担を減らし、報告の習慣化を後押しします。
- **手順書・ナレッジの標準化**：ISO形式の手順書テンプレートを使うことで、書き手による品質のばらつきを抑えられます。

研修の場面では、受講生に「書ける所だけ入力してみる」ところから始めてもらうと、心理的なハードルが下がり、文書作成への苦手意識をやわらげる効果が期待できます。

## 安全に使うために

業務文書を扱うツールであるため、情報の取り扱いには配慮が必要です。本ツールは、次のような設計になっています。

- **入力は端末内のみ**：入力した内容はブラウザのメモリ上にのみ保持され、外部に送信・保存されません（ページを再読み込みすると消えます）。
- **外部送信はAI利用時のみ**：AI整形を使ったときに限り、本文がAIサービスへ送信されます。それ以外の作成・保存はすべて端末内で完結します。
- **AIキーは保存しない**：AIを使う際は利用者自身のAPIキーをその場で入力する方式で、キーは保存されません。

運用上のお願いとして、**AI整形を使う際は、本文に実名・個人情報・機密情報を入力しない**ようご注意ください。同僚に言及する場合は役職名で記載する、といった運用を推奨します。生成された文章は、提出・公開の前に必ず内容をご確認ください。

## 技術的な裏側（概要）

本ツールは、フレームワークを使わず、HTML・CSS・JavaScriptを1ファイルにまとめて作られています。外部のライブラリ（CDN）を読み込まないため、ネットワークがない環境でも動作し、ファイル1つで配布・確認ができます。

設計上の特徴として、「どんな文書を・どんな項目で作るか」を**定義データ**として宣言しておく方式を採っています。この方式により、新しい文書タイプを追加したい場合も、原則として定義データを追記するだけで対応できる構造になっています。詳しい設計の解説は、別途の技術記事で取り上げます。

## まとめ

「報告書＆手順書ファクトリー PRO」は、**「型を知らなくても、型どおりの文書が出る」**ことを目指したツールです。

- 項目を埋めるだけで、14種の報告書・手順書が完成
- インストール不要・オフライン対応・5形式で書き出し
- AIは任意。端末内で完結する安全な設計
