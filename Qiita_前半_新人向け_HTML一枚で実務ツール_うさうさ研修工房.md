# 【前半・新人向け】フレームワークなしのHTML一枚で“実務ツール”を作る ― 報告書＆手順書ファクトリー PRO の作り方

> タグ: `JavaScript` `HTML` `初心者` `個人開発` `フロントエンド`

こんにちは、うさうさ先生🐰です。

この記事は、空欄を埋めるだけで報告書・手順書が完成する自作ツール「**報告書＆手順書ファクトリー PRO**」を題材に、**フレームワークもビルドも使わず、HTML一枚で実務ツールを作る**方法を解説するシリーズの **前半（新人向け）** です。

前半では「全体像」と「動く仕組みの土台」を、図と最小限のコードでやさしく解説します。後半（中堅向け）では、文書生成エンジン・エクスポート・AI連携・設計の良し悪しまで踏み込みます。

:::note info
**この記事で分かること（前半）**
- HTML一枚でアプリが動く、という素朴な作りの全体像
- 「定義データ → 状態 → 画面 → 文章」という一本道の考え方
- 入力するたびに画面が変わる「たった1つのループ」
:::

## 🐰 対象読者

- JavaScript を少し触ったことがある新人エンジニア
- 「自分でも小さなツールを作ってみたい」と思っている方
- React などの前に、素のJSで“動く仕組み”を理解したい方

## 🎯 まず、何ができるツール？

画面は3つのエリアだけです。

- **左**：項目を書き込むフォーム（件名・結論・実施内容…）
- **右**：入力に合わせて完成文がリアルタイムで組み上がるプレビュー
- **下**：Word / PDF / Markdown / CSV / Excel に書き出すボタン

報告書11種・手順書3種の **計14種** に対応し、各項目には「なぜ書くか」のヒントが付いています。難しい話は出てこないので、安心してついてきてください。

:::note info
🐣 **新人向けポイント**
このツールには **サーバーもデータベースもありません**。ブラウザで開く1枚のHTMLが全部です。あなたが入力した文字は、その画面のメモリの中だけに入り、ボタンを押すとファイルになって手元に保存されます。
:::

## 🏗️ 全体像：登場人物は5つだけ

コードを読むと、役割がきれいに5つに分かれています。まずこの5人を覚えると、地図ができます。

```text
① 定義データ  「どんな文書・項目があるか」を宣言
      │ 参照
      ▼
③ 画面づくり  定義を見て入力欄や見出しを並べる
      │ 入力(input)
      ▼
② 状態 store  入力した文字を1か所にためる
      │
      ▼
④ 文書づくり  状態を決まった順に並べて文章にする
      │
      ▼
⑤ 保存       完成文を Word / PDF などに書き出す
```

### 🍜 たとえ：ラーメン店モデル

- ①メニュー表（作れる種類）
- ②注文票（あなたが書く）
- ④厨房のレシピ（決まった順に並べる）
- 提供（右のプレビュー）
- ⑤持ち帰り（保存）

難しい処理はなく、**入口 → 処理 → 出口の一本道**の繰り返しだと思って大丈夫です。

## 📦 ①定義データ：作れるものを“宣言”する

「どんな文書を・どんな項目で作るか」を、最初にデータとして宣言しておきます。たとえば社内報告の項目は、こんなイメージです。

```js
// 「件名」「結論」などをデータとして宣言しておく
const FIELDS = {
  report: {
    naibu: [
      { key: "title", type: "text",     label: "件名",   tip: "一目で用件が分かるように" },
      { key: "concl", type: "textarea", label: "結論",   tip: "先に言い切る", out: "para" },
      { key: "acts",  type: "textarea", label: "実施内容", out: "bullets" },
      // …
    ]
  }
};
```

この「宣言（データ）」を画面づくりと文章づくりが読み取って、入力欄も完成文も自動で組み立てます。**コードであれこれ分岐させるのではなく、データを並べておく**——これが後半でも効いてくる、大事な考え方です。

## 📥 ②状態（store）：入力を1か所にためる

入力した文字は、`store` という1つの箱（オブジェクト）にためます。キーは「レッスン.タイプ.項目名」の形にして、文書タイプを跨いでも値が混ざらないようにしています。

```js
const store = {};
const ck   = k => lesson + "." + variant + "." + k;  // 名前空間つきキー
const getV = k => store[ck(k)] ?? "";                // 取り出す
const setV = (k, v) => { store[ck(k)] = v; };        // しまう
```

:::note info
🐣 **新人向けポイント：値はここだけ**
「あれ、入力した値ってどこにあるの？」と迷ったら、答えは `store` だけです。値はここにしか無いので、バグも基本ここを見れば追えます。タブやタイプを切り替えても消えないのは、この箱に残っているからです。
:::

## 🔄 ③④：たった1つのループ

このツールの心臓部は、**「入力 → 状態を更新 → 全部作り直す → 右に出す」**という1つのループです。1文字打つたびに、これが回っています。

```text
入力する（'input' イベント）
   │
   ▼
setV(key, value)        … store に保存
   │
   ▼
render() → buildDoc()   … 状態を読んで完成文を作り直す
   │
   ▼
#output.textContent = 完成テキスト   … 右のプレビューを更新
```

各入力欄には、こんなふうにイベントが結びついています。

```js
input.addEventListener("input", () => {
  setV(field.key, input.value);  // 箱を更新
  render();                      // 全部作り直して右に出す
});
```

「打つ → 箱に入る → 全部作り直す → 右が変わる」。ずっとこれの繰り返しです。難しそうに見えるツールも、分解するとこの一本道に収まります。

## ✨ 空欄はどうなる？

入力していない項目は、出力に出ません。文章を組み立てる関数が、値が空のときは見出しごとスキップするからです。

```js
function sec(L, label, val, mode) {
  if (!val) return;            // 空ならスキップ（見出しも出さない）
  L.push("■ " + label);
  // …本文を積む…
}
```

だから新人の方でも「書ける所だけ書いて、空欄は飛ばす」でOKです。空の見出しだけが残って不格好、ということが起きません。

## 🎁 前半のまとめ

- このツールは **HTML一枚＝アプリ**。サーバーもDBも無い、素朴な作り
- 中身は **「定義データ → 状態 → 画面 → 文章 → 保存」** の一本道
- 心臓部は **「入力 → store更新 → 全部作り直す → 出す」** のたった1つのループ
- **値は store の1か所だけ**／**空欄は出力に出ない**

ここまでで「動く仕組みの土台」がつかめたはずです。
